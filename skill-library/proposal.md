# /proposal

## Purpose
Draft a commercial proposal section for a BFSI prospect, in Predixion AI's standard structure and outcome-first framing, rather than a generic sales template.

## Context
You are drafting a commercial proposal for a BFSI prospect evaluating a Predixion AI product (collections automation, voice AI agents, deployment review tooling). The audience is typically a risk, collections, or ops decision-maker evaluating vendor options — they care about measurable outcomes and implementation risk, not feature lists.

## Input Required
- Prospect name, segment (NBFC, housing finance, lending, etc.)
- Product(s) being proposed
- Pricing model and figures, if finalized
- Known objections or concerns raised so far
- Any deployment timeline constraints

## Output Format
- **Outcome summary:** 2-3 bullets, framed as business outcomes (recovery rate, connect rate, cost-per-resolved-account) not feature descriptions
- **Pricing table:** clear unit basis (per seat / per account / per call), with what's included at each tier if there are tiers
- **Implementation timeline:** stage-by-stage (UAT, pilot, full rollout), each with a rough duration
- **Addressing known objections:** if input flags a specific concern, address it directly and specifically — don't write generic reassurance
- **Tone:** confident and outcome-based, not defensive or apologetic about pricing

## Constraints
- Never fabricate pricing, timelines, or outcome figures not provided in the input — flag missing inputs explicitly instead of inventing plausible-sounding numbers
- Use Indian English spelling conventions and BFSI-standard terminology (DPD buckets, NPA, recovery rate) precisely
- Keep each section scannable — assume the reader is comparing this against at least one competitor proposal
